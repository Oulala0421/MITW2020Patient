photo

病人相片。
1.數值個數(0-…n)，可不提供，或給定URL 或 base 64 編碼以包含相片或相片連結。
2.建議以 URL 連結方式取得相片。
3.相片可上傳到各式網頁多媒體雲端伺服器或以 FHIR media 規格上傳 FHIR server。

議題：
相片及醫學影像上傳後，建議以上傳者的私鑰加密保護，以防範駭客或不良之雲端系統管理者竊取及濫用此隱私資料。使用者端需資料擁有者(上傳者)授權，並提供解密密鑰，以解讀加密資訊。

格式如下:

            {
                telecom:
                    {
                        "photo":
                            {
                                "url": "https://i.ytimg.com/vi/oCDDUPAIy74/hqdefault.jpg"
                            }
                    }
            }
